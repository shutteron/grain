## MVPに追加すべき優先教材

### 優先度A：既存教材に統合してコンテンツを充実

1. **カメラ任せから卒業する** ← 既存あり、コンテンツ拡張
2. **写真は「意図」で変わる** ← 新規・または既存と統合
3. **露出とは何か** ← 既存あり、実践的な説明を追加
4. **ISO感度とは** ← 既存あり、現場での判断を追加
5. **F値とは** ← 既存あり、背景効果以外の側面を追加
6. **シャッタースピードとは** ← 既存あり、動きの表現を強化
7. **ホワイトバランスとは** ← 既存あり、空気感の表現を追加
8. **ダイナミックレンジとは** ← 新規で必要
9. **写真が暗い時の考え方** ← 既存（暗い顔）と統合
10. **顔が暗くなる理由** ← 既存あり、対策をより詳しく
11. **光と影の基本** ← 既存あり、立体感の理解を深める
12. **順光・逆光・サイド光** ← 既存あり、学べる要素を追加
13. **ストロボは何のために使うのか** ← 既存あり、「光設計」の概念を強化

## GRAINらしい言い換え方針

PDFの内容をGRAIN教材に落とし込む時の言語化の方向性：

| 従来の専門用語 | GRAINでの言い換え | 理由 |
|---|---|---|
| 露出 | 明るさの正解ではなく、写真の見せ方を決める設定 | 「正解」を求めるのではなく、「表現」であることを強調 |
| F値 | 背景ボケだけでなく、主役の深さを決める設定 | ボケは副産物、本質は光量と被写界深度の制御 |
| ISO感度 | 暗い場所での「救済手段」。上げるほどノイズが増えるトレードオフ | 必要性と限界を同時に理解 |
| シャッタースピード | ブレを止めるか、動きを残すかを決める設定 | 選択肢があること、表現の自由度を強調 |
| ホワイトバランス | 色の正確さだけでなく、写真の空気感を変える設定 | 色温度が感情に与える影響 |
| 逆光 | 失敗ではなく、透明感や輪郭を作れる光 | ネガティブからポジティブへの転換 |
| ストロボ | 暗い場所で光らせる道具ではなく、光を設計する道具 | 目的の本質的な理解 |
| 反射光測光 | カメラが光を測って露出を判断する仕組み（TTL測光） | 仕組みの理解で失敗への対処法が見える |
| 入射光測光 | 現場の光を直接測る手法（グレーカード、露出計） | より正確で再現性のある手法 |
| 18％グレー | 露出判断の基準となる「標準的な明るさ」 | 基準が存在することの重要性 |
| 測光 | 「どこの」「何の」「どの程度の」明るさを測るか | 測光の3要素を明確化 |
| ダイナミックレンジ | 写真に記録できる明るさの幅 | カメラの能力の理解、対策の必要性 |
| ボケ（被写界深度） | 主役に視線を集める表現手段 | ボケは手段であり、目的ではない |
| パース（パースペクティブ） | 焦点距離で被写体の見え方が変わる | 焦点距離による物理的な変化 |
| 圧縮効果 | 背景を整理する時に活躍する焦点距離の特性 | 具体的な活用場面を示す |
| TTL発光 | ストロボが自動で光量を調整する仕組みと限界 | 限界を知ることで対策が立てられる |
| ガイドナンバー | ストロボの光の強さを表す指標 | 指標の意味と活用法 |
| 照射角 | ストロボの光の広がりをコントロール | 焦点距離に合わせた最適化 |
| ストロボ同調速度 | ストロボ光が有効なシャッタースピードの上限 | 仕組みを理解すると対策が見える |
| ハイスピードシンクロ | 通常の同調速度を超えて撮影する技法 | 可能性の拡張、日中シンクロの実現 |
| ソフトボックス | 均一で柔らかい光を作るストロボ用具 | 光の質が立体感に与える影響 |
| フィルインライト | メインライトの影を和らげる補助光 | 光の役割分担の理解 |

## 編集のポイント

### PDFから学ぶべきこと、使わない方がいいこと

**学ぶべき：**
- テーマの優先順位と分類
- 初心者がつまずきやすいポイント
- 現場での実践的な判断
- 失敗パターンと対策

**使わない方がいい：**
- 図版やデザイン（GRAINは文章ベース）
- 複雑な技術説明（初心者向けに言い換え）
- PDFの事例写真（自分たちの実例を作成）
- 教科書的な文体（会話的に）

### GRAINの教材には以下の要素が必須

1. **beginnerExplanation** - 入門者にも分かる説明（なぜ？から始まる）
2. **fieldUse** - 現場での実際の使い方（どうやって？）
3. **commonMistakes** - 初心者がよくやる失敗（3～5 個）
4. **junichiNote** - じゅんいちさんの現場メモ（プロの視点）
5. **practice** - 実践練習課題（すぐに試せる）
